# 方針
- 無難かつ(ライブラリ的には)複雑でないナンカイイカンジの構成にする
  - ライブラリなどは可能な限りメジャーかつシンプルな形に留める
  - 作り方やプログラムは好き放題やる

# 採用ライブラリ
- cysharp.unitask
  - 非同期処理用のライブラリ。
  - ないと辛いし使ってみたいとのことだったし採用。他に代用知らないかも。
- cysharp.r3, observablecollections.r3
  - メッセージングライブラリ。
  - 何かしらはないと辛く、一番メジャーだし採用
- DoTween
  - アニメーション、トゥウィーンライブラリ。
  - 8割ぐらいで使うので入れとく
- UIEffects
  - UI Image用のエフェクト詰め合わせ。
  - 8割ぐらいで使うので入れとく

# スクリプト設計
Unity側の演出などの実装はMonoBehaviour、それ以外の制御ロジックはpure c#でかき分ける。
ジャム向きかと言われると怪しいけどまぁ、ジャムで使って悪いわけではないので

制御の流れ→依存性の流れ。フォルダ構造もそれに合わせる

また、それぞれ5つの大項目に分割する
- General
    - 全シーンで使うもの。カメラやトランジションなど
- Title
- Gatya
- Home
- (Root/Main)
    - ざっくりその他。臭いものがでたらこいつに押し付けてけ
    - データ管理系あれば分割するかも

一旦battleは考えない

# フォルダ分け
- Game
  - そのゲームの仕様に依存
  - unity非依存
- GameUnity
  - そのゲームの仕様に依存
  - unity依存(pure c#)
- Lib
  - そのゲームの仕様に非依存
  - unity非依存
- LibUnity
  - そのゲームの仕様に非依存
  - unity依存(pure c#)

それ以外はファイルの機能ごとに分けていく。
ファイルを切り分ける基準は役割が適切か。
